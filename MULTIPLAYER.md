# F1 Racing - Multiplayer Setup

Мультиплеер реализован с P2P архитектурой (один игрок — хост).

## Что реализовано

✅ **Меню система**
- Главное меню (Single Player / Multiplayer)
- Создание лобби
- Присоединение к лобби по коду
- Настройка количества кругов (3/5/10)

✅ **Signaling сервер**
- Управление комнатами
- WebRTC signaling для P2P соединений
- Поддержка 2-4 игроков

✅ **Network layer**
- P2P соединения через WebRTC
- Клиент-хост архитектура
- Интерполяция для плавного движения

✅ **Shared protocol**
- Типы сообщений
- Константы физики
- Общие типы данных

## Что осталось сделать

🔨 **Host game simulation** (Task #3)
- Хост запускает полную физику Rapier для всех игроков
- Применяет input от гостей
- Рассылает снапшоты состояния

🔨 **Guest game client**
- Получает снапшоты от хоста
- Интерполирует движение других машин
- Отправляет свой input

## Запуск для тестирования

### 1. Установить зависимости (уже сделано)
\`\`\`bash
npm install
\`\`\`

### 2. Запустить signaling server
\`\`\`bash
npm run dev:server
\`\`\`

Сервер запустится на `ws://localhost:3001`

### 3. Запустить клиент
В другом терминале:
\`\`\`bash
npm run dev
\`\`\`

Откроется браузер на `http://localhost:5173`

### 4. Тестирование мультиплеера

**Создать лобби:**
1. Нажми "Multiplayer"
2. Нажми "Create Lobby"
3. Введи имя
4. Скопируй код комнаты (6 символов)

**Присоединиться:**
1. Открой второе окно браузера (или инкогнито)
2. Нажми "Multiplayer" → "Join Lobby"
3. Введи имя и код комнаты
4. Нажми "Join"

**Запустить гонку:**
- Хост выбирает количество кругов
- Хост нажимает "Start Race"
- Начинается обратный отсчёт (пока только в консоли)

## Дебаг

Все сетевые события логируются в консоль с префиксами:
- `[Signaling]` — signaling сервер
- `[Network]` — network client
- `[Main]` — main.ts

Открой DevTools (F12) чтобы видеть что происходит.

## Известные ограничения

- Если хост отключится, гонка прервётся (нет host migration)
- Нет reconnect логики
- WebRTC может не работать через некоторые файрволы (нужны TURN серверы для продакшена)
