# Исправление видимости игроков в мультиплеере

**Дата:** 2026-05-04  
**Проблема:** Игроки не видят друг друга в мультиплеере

## Изменения

### 1. RacingState.ts - Исправлен расчёт позиции спавна
**Метод:** `getOpponentSpawnPosition()`

**До:**
```typescript
const opponentIndex = this.roomInfo?.players.findIndex(p => p.id === opponentId) ?? 0;
const offset = 1.5 + (opponentIndex * 2);
```

**После:**
```typescript
const guestPlayers = this.roomInfo?.players.filter(p => p.id !== this.playerId) ?? [];
const guestIndex = guestPlayers.findIndex(p => p.id === opponentId);
const offset = 1.5 + (guestIndex * 2);
```

**Причина:** Хост считал индекс среди всех игроков (включая себя), что давало неправильный offset. Теперь считается только среди гостей.

### 2. OpponentController.ts - Добавлено логирование

**Метод:** `addRemotePlayer()`
- Логирование создания меша
- Предупреждение если игрок уже существует
- Вывод позиции меша и общего количества оппонентов

**Метод:** `updateRemotePlayer()`
- Предупреждение если снапшот для неизвестного игрока

**Метод:** `updateRemoteVisuals()`
- Предупреждение если нет данных интерполяции
- Предупреждение если ни один оппонент не обновлён
- Счётчик обновлённых оппонентов

### 3. RacingState.ts - Улучшено логирование инициализации

**Метод:** `initializeMultiplayerOpponents()`
- Вывод режима игры (host/guest)
- Счётчик созданных мешей для гостя
- Количество буферизованных снапшотов

## Ожидаемое поведение

### Хост (multi_host)
1. Создаёт физическое тело для гостя на позиции baseX + 1.5
2. НЕ создаёт визуальный меш через OpponentController (использует физический меш)
3. Отправляет снапшоты гостю с позициями обоих игроков

### Гость (multi_guest)
1. Создаёт визуальный меш для хоста через OpponentController
2. Устанавливает начальную позицию хоста на baseX - 1.5
3. Получает снапшоты от хоста и обновляет позицию через интерполятор
